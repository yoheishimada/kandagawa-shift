毎日の使い方

  1. 日付を選ぶ
    画面左のパネルから、計画を立てたい週の開始日を選びます。
  2. モードを選ぶ
    その日の状況にあわせて選んでください。
    🔴
    強気
    天気がよい
    イベントがある
    🟢
    普通
    いつも通りの日
    🔵
    弱気
    雨・連休明け
    など
  3. 製造数を確認する
    商品ごとの製造推奨数が表示されます。スプレッドシートと同じ並び順なので、そのまま転記できます。

画面の見方

週間カード

1週間分の売上予測が並びます。数字は税込の予測売上です。

祝　祝日 学休　学校休み 早大休　早稲田休み 昼雨 / 夕雨　雨予報

製造計画

3つのセクションに分かれています。

  • 🍞 パン — 食パン・惣菜パンなど
  • 🥪 サンドイッチ — バゲットサンド・パニーノなど
  • 🔥 二次製品 — フレンチトースト・クロックムッシュなど
🔴 売切注意 — よく売り切れる商品。多めに検討
🔵 廃棄注意 — ロスが出やすい商品。少なめに検討

バッファーとは

製造数に上乗せする「余裕」のことです。

0%
予測通りに製造
10%
予測より1割多く製造 ← 通常はここ
20%
予測より2割多く製造
売切を避けたい日は多め、廃棄を減らしたい日は少なめに調整してください。

月に1回やること

前月のスプレッドシートが埋まったら実施します。月初めに約5分で完了します。

1
Macでターミナルを開く
Launchpadから「ターミナル」を検索
2
以下をコピーして貼り付け、Enterを押す cd "/Users/shimadayohei/Desktop/神田川ベーカリー売り上げアプリ" python3 parse_sheets.py python3 build_calibration.py スプレッドシートの実績を取り込み、ML補正係数を更新します
3
続けて以下を実行してGitHubにアップロード git add calibration.pkl sheet_actuals.pkl git commit -m "月次更新" git push pushすると自動でクラウドに反映されます（約2〜3分）
4
2〜3分後にアプリをリロードして確認
ブラウザで再読み込み（F5 または ⌘R）するだけです

困ったときは

アプリが開かない インターネット接続を確認してください。しばらく待ってから再度アクセスしてみてください
パスワードが通らない 大文字・小文字に注意して入力してください。コピー&貼り付けが確実です
天気が取れないと表示される 17日以上先は天気予報が使えません。予測は続きます
数字が古い気がする ブラウザを再読み込み（F5 または ⌘R）してください
月次更新後に反映されない git pushから2〜3分待ってからブラウザを再読み込みしてください
